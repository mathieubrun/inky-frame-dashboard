## Summary

Implement a mechanism to track Inky Frame battery levels. The device will report its current voltage to the API via a POST request (`/api/v1/battery`) before each dashboard update. The server will append this data (timestamp and voltage) to a local CSV file. A second endpoint (`GET /api/v1/battery/history`) will serve the raw CSV content. This approach minimizes device-side logic while providing persistent history for analysis.

## Technical Context

**Language/Version**: Go (Golang) 1.22+
**Primary Dependencies**: net/http, spf13/cobra, viper
**Storage**: Local CSV file (e.g., `data/battery.csv`)
**Testing**: standard library testing (Required, >80% coverage)
**Target Platform**: Raspberry Pi Pico W (Inky Frame) + Linux Server (API)
**Project Type**: Web Service + CLI
**Performance Goals**: API response time < 100ms for reporting; minimal impact on device wake-time.
**Constraints**: 
- Must use standard library `net/http` for API.
- CSV format: `date,voltage` (RFC3339 for date).
- Must adhere to "Energy-First Lifecycle" (minimal Wi-Fi on-time).
**Scale/Scope**: Single device tracking, extensible to multiple devices if needed (FR-003).
